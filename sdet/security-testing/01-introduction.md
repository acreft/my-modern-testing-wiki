# Introducción

El Security Testing busca garantizar que el software protege los datos y funciones frente a accesos no autorizados y amenazas maliciosas.
En la pirámide de pruebas de ISTQB®, se considera un tipo de prueba no funcional, pero crítico en aplicaciones modernas.

Su alcance incluye tanto aspectos técnicos (vulnerabilidades en código, infraestructura, APIs) como aspectos organizativos (roles, accesos, cumplimiento de normativas como GDPR o PCI-DSS).

# Tipos de pruebas

- Vulnerability scanning → identifica debilidades conocidas.

- Penetration testing → simula ataques reales.

- Autenticación/autorización → evalúa login, permisos, gestión de sesiones.

- Cifrado y almacenamiento → revisa seguridad de datos sensibles.

# Marcos de referencia

- OWASP Top 10 → riesgos en aplicaciones web (SQL Injection, XSS, CSRF).

- OWASP Mobile Top 10 → riesgos en apps móviles.

- NIST Cybersecurity Framework.

# Herramientas comunes

- OWASP ZAP, Burp Suite (escaneo y análisis).

- Nessus (vulnerabilidades).

- MobSF (apps móviles).

# Buenas prácticas

- Integrar seguridad desde fases tempranas (Shift Left Security).

- Automatizar análisis estático (SAST) y dinámico (DAST).

- Revisar dependencias externas (Software Composition Analysis).
